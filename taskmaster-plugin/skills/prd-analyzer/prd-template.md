# Condensed PRD Template (Reference)

## Core Sections

1. **Document Info**: Version, owner, status
2. **Executive Summary**: What & why (2-3 paragraphs)
3. **Problem**: Current pain points
4. **Goals**: Business & user objectives with KPIs
5. **Users**: Primary/secondary personas with pain points
6. **Features**: Priority (P0-P3), user stories, acceptance criteria, tasks
7. **Technical**: Architecture, APIs, data models
8. **Timeline**: Phases, milestones, dates
9. **Testing**: Strategy, scenarios, coverage targets

## Task Structure Format

```
### Feature X: [Name] [P0/P1/P2]

**User Story**: As [user], I want [goal], so that [benefit]

**Acceptance Criteria**:
- [ ] Criterion 1
- [ ] Criterion 2

**Tasks**:

#### Task X.1: [Name]
Estimated: Xh | Dependencies: [IDs] | Priority: PX

**Subtasks**:
- [ ] X.1.1: [Description] (Xh)
- [ ] X.1.2: [Description] (Xh)
```
